# Software-Engineering-Spring-2026
This repository documents my contributions to the Software Engineering course at Constructor University. The curriculum is meticulously designed to mirror a real-world software engineering environment, moving beyond academic theory to simulate the high-stakes, collaborative nature of the industry.

The "Industry-Standard" Relay Cycle

- In professional software development, you rarely start a project from scratch or stay on one team forever. This course replicates that reality through a bi-weekly rotation system:

- Two-Week Sprints: Every 14 days, I am paired with a new partner and assigned to a completely different project repository.

- Rapid Onboarding: Like joining a new company or department, I must quickly understand a foreign codebase, identify technical debt, and push the project toward its final goal within a strict deadline.

- The Handover: After the sprint, the team and repository change. I pass my work to a new group, requiring my code to be production-ready and easily digestible for my successors.

Focus: Maintainability & Clean Code

Because this project is a "living" codebase that changes hands frequently, maintainability is the primary metric of success. To ensure the next team can hit the ground running, I focus on:

- Self-Documenting Code: Writing clean, modular logic in C++ that explains why a decision was made, not just what it does.

- Standardized Documentation: Maintaining clear READMEs and architectural diagrams to minimize the "knowledge gap" during transitions.

- Sustainable Development: Balancing the urge to add new features with the necessity of refactoring, ensuring the project remains stable for the duration of its lifecycle.
