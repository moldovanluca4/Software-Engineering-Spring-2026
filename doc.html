Project 3 Client Documentation


Name

client -- connect to bug world sim and allow user to input commands


Synopsis

client world_map bug1 bug2 [ticks_per_frame] [fps]

ArgumentRequiredDescriptionDefault
world_mapYesPath to a .world file--
bug1YesPath to a .bug file (Red team program)--
bug2YesPath to a .bug file (Black team program)--
ticks_per_frameNoNumber of simulation ticks per display frame50
fpsNoTarget frames per second for display refresh10

Description

client connects to bug world sim and allows user to input commands. client starts the sim with the given .world and .bug files (sim doc contains info about file formats and output) as a subprocess, then allows the user to type in commands that can affect the sim, this is currently done through the terminal for testing purposes. The ouput of the sim is also currently shown on the terminal. client communicates with sim through cmd_pipe which is used to send commands and data_pipe which is used to read output from the sim. The correct order to open the pipes for client is:

  1. Open the cmd pipe for writing
  2. Open the data pipe for reading

Commands are plain text, one per line, terminated by \n.

CommandDescription
FETCH\nReturn the current world state without advancing
STEP N\nAdvance N simulation ticks, then return the state
QUIT \nShut down the simulator cleanly

Files

client depends on the sim.asm file used to simulate bug world as well as the file.world and the bug1.bug and bug2.bug files.


Exit code

Upon succesful completion, client exits with exit code 0. On failure, the error is printed to cerr and exit code 1 is returned.
